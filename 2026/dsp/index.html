review遮盖法求并行滤波器

很久之前我在steepflanger里用过这种方式计算并行的切比雪夫I型滤波器，但现在我看不懂了，反正要重新学，这次就研究的更加透彻一点。

让我们先来一个简单的数学问题，给定 $$ f(x)=\frac{1}{x-1}\frac{1}{x-3} $$

求 $$ f(x)=\frac{A}{x-1}+\frac{B}{x-3} $$中的A和B。

接下来是使用遮盖法计算的过程

$$ A=f(1)(x-1)=\frac{1}{1-3}=-\frac{1}{2} $$

$$ B=f(3)(x-3)=\frac{1}{3-1}=\frac{1}{2} $$

验证一下，在desmos里绘制出来，完全一致，正确。

这是怎么运行的，哈基米给我解释一下

**遮盖法（Heaviside Cover-up Method）**的本质并不是什么凭空出现的“魔法”，它其实是**代数代入法**的一种高度简化的捷径。

为了让你一眼看透它的工作原理，我们直接还原它“被遮住”之前的真实代数推导过程。

---

### 核心原理：代数消除法

我们依然以 $f(x) = \frac{1}{(x-1)(x-3)} = \frac{A}{x-1} + \frac{B}{x-3}$ 为例。

#### 1. 为什么求 $A$ 时要“遮住” $(x-1)$？

我们的目标是求出未知数 $A$。在代数中，想要孤立 $A$，最直接的方法是**让等式两边同时乘以 $(x-1)$**：

$$\frac{1}{(x-1)(x-3)} \cdot (x-1) = \left( \frac{A}{x-1} + \frac{B}{x-3} \right) \cdot (x-1)$$

约分简化后，等式变成了：

$$\frac{1}{x-3} = A + \frac{B(x-1)}{x-3}$$

#### 2. 为什么要代入 $x = 1$？

现在看等式右边，除了 $A$ 以外，还拖着一个含有 $B$ 的尾巴 $\frac{B(x-1)}{x-3}$。
为了**干掉 $B$ 的干扰**，最聪明的方法就是让分子的 $(x-1) = 0$，也就是令 **$x = 1$**。

将 $x = 1$ 代入整张等式：

$$\frac{1}{1-3} = A + \frac{B \cdot (1-1)}{1-3}$$

$$\frac{1}{-2} = A + 0$$

$$A = -\frac{1}{2}$$

good，现在我了解了。接下来来看下分子和分母阶数相同的该如何求解常数项。

当分子和分母的**阶数（次数）相同**时，这个分式属于**假分式**。

如果强行直接使用遮盖法，你会发现少了一个常数项（即你说的**直接项**，也就是当 $x \to \infty$ 时的极限值）。这是因为遮盖法只能提取出分母各个因式对应的*真分式*部分。

要解决这个问题，有两种最主流、最快的处理策略：

---

### 方法一：直接项口诀——最高次系数比（最快）

当分子分母阶数相同时，直接项（常数项）永远等于**分子最高次项的系数 除以 分母最高次项的系数**。

#### 💡 核心公式：

设 $f(x) = \frac{a_n x^n + \dots}{b_n x^n + \dots} = \text{直接项} + \frac{A}{x-x_1} + \frac{B}{x-x_2} + \dots$
则：

$$\text{直接项} = \frac{a_n}{b_n}$$

#### 举个例子：

假设函数为：

$$f(x) = \frac{2x^2 - 5x + 1}{(x-1)(x-3)}$$

1. **求直接项：** 分子最高次项是 $2x^2$，系数为 `2`；分母展开后最高次项是 $x^2$，系数为 `1`。
所以 **直接项 = 2 / 1 = 2**。
2. **其余项继续用遮盖法：**
我们将式子设为：$\frac{2x^2 - 5x + 1}{(x-1)(x-3)} = 2 + \frac{A}{x-1} + \frac{B}{x-3}$
* 求 $A$（遮盖 $x-1$，代入 $x=1$）：
$$A = \left. \frac{2x^2 - 5x + 1}{x-3} \right|_{x=1} = \frac{2(1)^2 - 5(1) + 1}{1-3} = \frac{-2}{-2} = 1$$

* 求 $B$（遮盖 $x-3$，代入 $x=3$）：
$$B = \left. \frac{2x^2 - 5x + 1}{x-1} \right|_{x=3} = \frac{2(3)^2 - 5(3) + 1}{3-1} = \frac{18 - 15 + 1}{2} = \frac{4}{2} = 2$$

最终展开结果：

$$f(x) = 2 + \frac{1}{x-1} + \frac{2}{x-3}$$

---

### 方法二：凑项法（适合心算）

如果你不想背公式，可以通过在分子上**加一项、减一项**来人为创造出分母的倍数，从而直接“剥离”出直接项。

依然用上面的例子 $f(x) = \frac{2x^2 - 5x + 1}{x^2 - 4x + 3}$：

1. 观察分母是 $x^2 - 4x + 3$，分子最高次是 $2x^2$。
2. 我们在分子凑出 `2 * 分母` 的形式，即 $2(x^2 - 4x + 3) = 2x^2 - 8x + 6$。
3. 强行改写分子：

$$\frac{2x^2 - 5x + 1}{x^2 - 4x + 3} = \frac{(2x^2 - 8x + 6) + 3x - 5}{x^2 - 4x + 3}$$

$$= \frac{2(x^2 - 4x + 3)}{x^2 - 4x + 3} + \frac{3x - 5}{(x-1)(x-3)}$$

$$= 2 + \frac{3x - 5}{(x-1)(x-3)}$$

现在，前面的 **`2` 就是直接项**，而后面的 $\frac{3x - 5}{(x-1)(x-3)}$ 已经变成了标准的真分式。对其再次使用传统的遮盖法处理，即可同样得到 $A=1, B=2$。

好了，到现在我们就学会了遮盖法。面对并行滤波器分解这种问题就简单多了，只不过是把x替换为z或者z^-1，根可能是复数罢了（不考虑重复极点）

有趣的是，以z或者z^-1来进行求解会得到两套不一样的滤波器

以z求解，会得到常数项=1（如果有的话），每个并行滤波器都需要延迟一个采样

以z^-1求解，会得到没有延迟的滤波器，但常数项需要求解

根据哈基米的说法，以z求解在数值上更稳定（我不知道，问问它引用和为什么）

超采样器

多相滤波器上采样下采样效率高高，支持任意整数倍。音频开发里有什么用到上下采样了呢，失真，那失真是用多相滤波器做么？

用到了，但不完全是。

让我们看看实际工程都在怎么做

https://github.com/ZL-Audio/ZLCompressor/tree/main/source/dsp/over_sample

我们看到一个滤波器系数就能支持任意2^N的超采样系数，多相滤波器里不是这样的，你应该每次设计pi/M的滤波器系数才对。

那么大佬这个是咋工作的呢。

可以看到超采样是由好几个2X超采样的滤波器叠起来了，这也就意味着只能使用2^N的超采样系数。

难怪平常的插件里超采样系数都是2x 4x 8x这种形式，原来大家都在这么做。

剩下的就是一些优化小技巧，比如使用半带FIR滤波器，截止频率在pi/2，且长度为奇数。这使得从中心开始，往两边+2 -2，例如+2 +4 +6 +8 -2 -4 -6 -8的位置都非常接近0.

实际上，我们可以直接把这些位置设为0，损失了一点精确的幅度响应，但这可以让我们使用牛逼的优化技巧：这些零不需要计算。

配合上多相分解的技巧，even位算half_len个数据，odd位就只需要计算一个数据，就是中心位置。

class CombAllpass {
public:
    void  Init(float sampleRate);
    float Process(float in);
    void  ClearInternal();
    void  SetDelay(float delay);
    float GetDelay() const { return delay_.GetDelay(); }
    void  SetAlpha(float a);
    float GetAlpha() const { return alpha_; }
    float GetLast() const { return last_; }
private:
    DelayLine delay_;
    float alpha_{};
    float last_{};
};

class Allpass {
public:
    float Process(float in) {
        auto v = latch_;
        auto t = in - alpha_ * v;
        latch_ = t;
        return v + alpha_ * t;
    }
    void  ClearInternal() {
        latch_ = 0;
    }
    void  SetAlpha(float a) {
        alpha_ = a;
    }
private:
    float alpha_{};
    float latch_{};
};
                    
                
                    
void CombAllpass::Init(float sampleRate) {
    delay_.Init(sampleRate);
}

float CombAllpass::Process(float in) {
    auto v = delay_.GetLast();
    auto t = in - alpha_ * v;
    delay_.Push(t);
    last_ = v + alpha_ * t;
    return last_;
}

void CombAllpass::ClearInternal() {
    delay_.ClearInternal();
}

void CombAllpass::SetDelay(float delay) {
    delay_.SetDelay(delay);
}

void CombAllpass::SetAlpha(float a) {
    alpha_ = a;
}
